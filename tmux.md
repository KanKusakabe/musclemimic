tmuxの使い方まとめ
RunPodなどのGPUサーバーで、VS CodeやPCを閉じてもプログラムを裏で回し続けるためのコマンド集です。

1. 基本的な流れ（これだけ知っていればOK）
1. 新しく画面を作る（起動する）

```

tmux new -s myjob

```

2. プログラムを実行する

```

python main.py

```

3. 実行したまま自分のPCから離脱する（デタッチ）

  • キーボードの `Ctrl` + `B` を押したあと、指を離して `D` を押す

---

2. 後から様子を確認するとき
• 動いている画面の一覧を見る

```

tmux ls

```

• さっきの画面に再接続する（アタッチ）

```

tmux a -t myjob

```

(※セッションが1つだけなら `tmux a` だけでOK)

---

3. その他の便利コマンド
• tmuxの画面の中で、ログをスクロールして見たいとき

  1. `Ctrl` + `B` を押したあと、 `[` を押す（コピーモードに入る）

  2. 矢印キーやマウスホイールで過去のログを上下にスクロールできる

  3. スクロールを終わらせる時は `Q` キーを押す

• 作業が終わって画面を完全に消去（削除）したいとき

  • アタッチした状態で `exit` と入力して Enter を押す

---

💡 注意点
VS CodeのJupyter Notebook（`.ipynb`）のセル（▶ボタン）で実行した処理はtmuxの保護下に入りません。 裏で回し続けたい重い処理は、必ずアタッチした**ターミナル上（コマンド）**で実行してください。
